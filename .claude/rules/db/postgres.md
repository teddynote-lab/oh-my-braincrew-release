# Postgres Standards

## Schema Design

- Use `snake_case` for table and column names.
- Every table MUST have a primary key (`id`).
- Use `created_at` and `updated_at` timestamps on all tables.
- Use appropriate types: `uuid` for IDs, `timestamptz` for timestamps, `text` over `varchar` unless length matters.
- Add indexes for columns used in WHERE, JOIN, and ORDER BY clauses.
- Foreign keys with appropriate ON DELETE behavior (CASCADE, SET NULL, RESTRICT).

## Migrations (Alembic)

- One migration per logical change — don't combine unrelated schema changes.
- Migration file naming: auto-generated by Alembic (revision hash + description).
- Always test migrations forward AND backward (upgrade + downgrade).
- Never modify a published migration — create a new one.
- Destructive operations (drop column, drop table) require explicit confirmation.

```python
# alembic/versions/xxxx_add_users_table.py
def upgrade():
    op.create_table('users',
        sa.Column('id', sa.UUID, primary_key=True, server_default=sa.text('gen_random_uuid()')),
        sa.Column('email', sa.Text, nullable=False, unique=True),
        sa.Column('created_at', sa.DateTime(timezone=True), server_default=sa.text('now()')),
    )

def downgrade():
    op.drop_table('users')
```

## SQLAlchemy / asyncpg

- Use async engine and session for FastAPI: `create_async_engine`.
- Session per request via FastAPI dependency injection.
- Use `select()` / `insert()` expressions — not raw SQL strings.
- Close sessions properly — use `async with` or dependency cleanup.

## Connection Pooling

- Pool size = expected concurrent requests (typically 10-20).
- Set `pool_timeout` to fail fast rather than queue indefinitely.
- Monitor pool health: active connections, wait time.
- Close pool on application shutdown.

## Query Safety

- NEVER use string interpolation in queries.
- Always use parameterized queries or ORM expressions.
- Use `EXPLAIN ANALYZE` for slow queries — add indexes as needed.

## Anti-Patterns

- N+1 queries: use eager loading (`joinedload`, `selectinload`).
- Missing indexes on frequently queried columns.
- Overly broad `SELECT *` when only specific columns are needed.
- Storing JSON blobs when structured columns would be clearer.
